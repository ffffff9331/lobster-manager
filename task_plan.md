# openclaw-manager 性能收口计划

## 目标
在不扩功能的前提下，完成 openclaw-manager 正式版前的性能收口：定位启动慢、页面切换慢、首开重加载链路，并落最小可验证修复。

## 当前已知事实
- 项目当前口径：v2.0.0 Beta 已发布，正式版未完成
- 已知严重卡顿不是单点 UI，而是启动和页面切换阶段后台自触发加载过多
- 已确认的历史慢源包括：启动预拉多页数据、版本检查未设超时、useSystemState 启动自跑系统信息与更新检查、useChatChannels 启动自跑渠道状态、任务页首开顺手加载首个 cron run 记录
- 当前目标不是大改架构，而是继续按页懒加载、减少自动触发、给重操作加边界和超时

## Phase
- [complete] Phase 1: 盘点当前自动触发链路与慢源入口
- [complete] Phase 2: 设计最小修法并修改代码
- [complete] Phase 3: 构建验证 + 本地启动验证
- [in_progress] Phase 4: 输出本轮性能收口结论与剩余尾项

## 本轮验收标准
- 能明确列出当前仍存在的自动触发加载链路
- 至少落一轮最小性能修复，不是只停留在分析
- 构建通过
- 对修改影响给出真实结论，不夸大

## 本轮结果
- 已确认当前主要慢源之一是多个重页面在首次进入时仍保留自动刷新链路
- 已对 `tasks / models / skills / doctor / settings` 取消 `initialRefreshers`，避免这些页面首开即自动拉重数据
- 已保留更核心页面的保守自动刷新口径，不扩大为全局停更
- 已完成 `npm run build` 验证
- 仍需做一轮真实 Tauri 宿主环境体感验证，确认首开和切页是否明显改善
